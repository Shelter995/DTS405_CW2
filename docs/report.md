# Basketball Broadcast Detection and BEV Mini-map Report

## 1. Introduction

本次实验实现了一个篮球画面转播的 computer vision task。我首先使用 YOLOv8n 对比赛画面中的动态实体进行 multi-object detection，然后手动标定 planar homography 将图像坐标映射到标准篮球场 2D bird's-eye-view (BEV) mini-map 上，最后生成带检测框、tracking ID 和实时小地图的 demo video。

按照要求，实验没有做 3D reconstruction，但是完成一个可解释、可复现的 2D BEV tactical visualisation system。根据给定数据集，模型训练类别为 `player`, `ball`, `referee`, `others`。作业描述中提到 hoop，但数据标签中没有独立 hoop 类，因此实验根据实际已标注的label完成定量检测评估，并将战术小地图重点放在 player、referee 和 ball 的位置，并进行可视化。

## 2. Dataset and Pre-processing

原始数据包含一个 training clip 和三个 testing clips。每个 clip 内部包含 `images/`, `labels/`, `classes.txt`, `notes.json`。YOLO label 格式为：

```text
class_id x_center y_center width height
```

其中坐标均为 normalised image coordinates。项目将原始 clips 整理为标准 YOLO 目录结构，并保留原始类别顺序，避免 label/config class mismatch。

| Split | Source | Frames | Purpose |
|---|---:|---:|---|
| Train | `clip1` first 400 frames | 400 | YOLO fine-tuning |
| Validation | `clip1` last 100 frames | 100 | Model selection and validation metrics |
| Test | `clip2 + clip3 + clip4` | 1500 approx. | Generalisation evaluation |
| Demo | `data/Demo Video.mp4` | video frames | Final visualisation |

机位关系上，`clip1/clip4` 属于 Camera A，`clip2/clip3` 属于 Camera B。最终 demo 使用课程提供的 `data/Demo Video.mp4`。homography 标定直接基于该视频第一帧完成：我从第一帧中手动选择位于球场左侧线和中线附近的四个可见 court reference points，并将它们对应到标准全场篮球场模板。

## 3. Object Detection Method

### 3.1 YOLO Model Choice

本项目使用 `YOLOv8n` 作为 baseline detector。选择 nano model 的原因是：训练速度快、显存占用小、适合 coursework 场景下快速完成 detection → homography → mini-map rendering 的完整 pipeline。模型从 COCO pretrained weights `yolov8n.pt` 初始化，然后在篮球数据集的四个类别上 fine-tune。

主要训练设置如下：

| Parameter | Value |
|---|---:|
| Base model | `yolov8n.pt` |
| Epochs | 50 |
| Image size | 640 |
| Batch size | 16 |
| Seed | 42 |
| Confidence threshold for demo | 0.25, adjustable |
| Augmentation | disabled/minimised for interpretable baseline |
| Output weights | `outputs/runs/detect/yolov8n_noaug/weights/best.pt` |

训练过程中，YOLO 的 detection head 会从 COCO 的 80 classes 改为本数据集的 4 classes。日志中 `Transferred 319/355 items from pretrained weights` 表明 backbone/neck 的大部分参数被迁移，最后 detection head 因类别数变化而重新学习。

### 3.2 Detection Post-processing and Foot-point Extraction

对于每个检测到的 player，本项目计算 bounding box 的 bottom-centre coordinate `(u, v)` 作为脚点/地面接触点。这一步是从 image space 到 court plane 的关键接口。

如果 player bounding box 为：

```text
(x1, y1, x2, y2)
```

则 foot-point 定义为：

```text
u = (x1 + x2) / 2
v = y2
```

该点近似表示球员双脚与地面的接触位置。referee 也使用相同 bottom-centre 策略，因为裁判同样位于地面平面上。ball 则使用 bbox centre，因为篮球可能在空中，bottom-centre 并不稳定。

**Pseudocode: detection and foot-point extraction**

```text
for each frame:
    detections = YOLO.predict(frame)

    for each detection in detections:
        if class == player:
            x1, y1, x2, y2 = detection.xyxy
            foot_u = (x1 + x2) / 2
            foot_v = y2
            save player foot-point (foot_u, foot_v)

        if class == referee:
            x1, y1, x2, y2 = detection.xyxy
            foot_u = (x1 + x2) / 2
            foot_v = y2
            save referee foot-point (foot_u, foot_v)

        if class == ball:
            ball_u = (x1 + x2) / 2
            ball_v = (y1 + y2) / 2
            save approximate ball point
```

### 3.3 Tracking for Visualisation

最终 demo 中使用 lightweight tracking 为 player/referee 保持 temporal ID。实际渲染采用双路逻辑：

1. `model.predict()` 负责主画面 detection boxes，避免 tracker 过滤掉有效检测框。
2. `model.track()` 负责 player/referee ID、mini-map 点和短轨迹。
3. 通过 IoU matching 将 tracker ID 尽量匹配回 predict boxes，使主画面和小地图 ID 一致。

tracking 让小地图从“闪烁点云”变成可读的 temporal tactical map。

## 4. Manual Homography and Mini-map Projection

### 4.1 Why Homography

篮球场地面可以近似看作一个 planar surface。对于固定或近似固定 camera，地面上任意图像点 `p = [u, v, 1]^T` 与标准球场 BEV 坐标 `P = [x, y, 1]^T` 之间可以通过 3x3 homography matrix `H` 建立关系：

```text
P ~ H p
```

这里 `~` 表示 homogeneous coordinates 下的 scale equivalence。实际投影后需要除以第三维：

```text
[x', y', w']^T = H [u, v, 1]^T
x = x' / w'
y = y' / w'
```

### 4.2 Manual Reference Points

作业要求强调 manual implementation，因此本项目没有直接调用 `cv2.findHomography` 作为主方法，而是在 `Demo Video.mp4` 第一帧中手动选择 court landmarks。实际使用的四个参考点来自球场左侧线和中线附近的可见线段/交点，这些点在视频画面中相对清晰，且位于同一 ground plane，适合建立 image-to-court correspondence。参考点必须满足：

- 位于地面平面上，例如 sideline/intersection/free-throw line/key corner；
- 不选择球员、篮球、篮筐、广告牌等非地面点；
- 至少 4 点，且不能接近共线；
- 点的 image order 必须与 template court coordinate order 一致。

### 4.3 DLT/SVD Derivation

设图像点为 `(u, v)`，目标 BEV 点为 `(x, y)`，homography 为：

```text
H = [h1 h2 h3
     h4 h5 h6
     h7 h8 h9]
```

由 `P ~ H p` 可得：

```text
x = (h1*u + h2*v + h3) / (h7*u + h8*v + h9)
y = (h4*u + h5*v + h6) / (h7*u + h8*v + h9)
```

整理后，每一对 correspondence 产生两条 linear constraints：

```text
[-u, -v, -1,  0,  0,  0, x*u, x*v, x] h = 0
[ 0,  0,  0, -u, -v, -1, y*u, y*v, y] h = 0
```

多组点构成线性系统：

```text
A h = 0
```

本项目手动实现 DLT (Direct Linear Transform)，并使用 SVD 求解最小奇异值对应的 `h`。为提高数值稳定性，实现中先对 source/target points 做 normalisation，再反归一化得到最终 `H`。

**Pseudocode: manual homography**

```text
input:
    image_points = [(u1, v1), ..., (un, vn)]
    court_points = [(x1, y1), ..., (xn, yn)]

normalise image_points and court_points

A = []
for each pair ((u, v), (x, y)):
    append [-u, -v, -1, 0, 0, 0, x*u, x*v, x]
    append [0, 0, 0, -u, -v, -1, y*u, y*v, y]

U, S, Vt = SVD(A)
h = last row of Vt
H_normalised = reshape(h, 3, 3)
H = inverse(T_court) * H_normalised * T_image
H = H / H[2, 2]
```

### 4.4 Mini-map Rendering

目标模板采用标准横向 full-court layout，物理比例为 `28m x 15m`，再映射到 mini-map pixel canvas。实现中将 BEV projection canvas 与 video overlay display size 分离：homography 输出始终落在 `homography_camera_b.json` 中定义的固定模板坐标系上，右上角小地图只在最终叠加前做 resize。这样可以避免因为放大显示尺寸而改变投影坐标系。

最终视频中，小地图显示：

- player: enlarged blue circles with black external ID labels `P1`, `P2`, ...；
- referee: enlarged yellow/orange circles with black external ID labels `R1`, `R2`, ...；
- ball: red dot；
- player/referee short trajectories: last 20 frames, rendered in gold for better contrast against the court template；
- `others`: only shown on main frame, not projected to mini-map。

**Pseudocode: video rendering**

```text
open Demo Video.mp4
open output video writer

for each frame:
    predict_detections = YOLO.predict(frame)
    track_detections = YOLO.track(frame)

    draw all detection boxes on main frame

    for each tracked player:
        foot = bbox_bottom_centre(player_box)
        bev_point = H * foot
        update player_tracks[player_id]
        draw P-id on mini-map

    for each tracked referee:
        foot = bbox_bottom_centre(referee_box)
        bev_point = H * foot
        update referee_tracks[referee_id]
        draw R-id on mini-map

    if ball detected:
        centre = bbox_centre(ball_box)
        bev_ball = H * centre
        draw red ball point
    else:
        keep previous ball point for a few frames with fading

    resize mini-map for display
    overlay enlarged mini-map at top-right
    write combined frame to output video
```

```text
F1 = 2PR / (P + R)
```

Validation 结果明显高于 test，说明模型对 `clip1` 同场景有较好适应，但跨 clip/camera 后泛化下降。这个差距符合 basketball broadcast 数据的特点：不同 camera angle、player scale、lighting、occlusion pattern 都会改变检测难度。

## 6. Critical Analysis

### 6.1 Occlusion and Scale Variation

篮球比赛中球员经常集中在篮下或边线附近，bounding boxes 会相互重叠。YOLOv8n 在 player 类上仍能保持较高 recall，但在严重遮挡时可能出现 missed detection 或 ID switch。尺度变化也很明显：前景球员 bbox 大、远端球员 bbox 小，small distant players 更容易被漏检。

### 6.2 Ball Detection Difficulty

ball 是最困难的类别。test overall 中 ball recall 只有 0.003，说明几乎所有篮球实例都被漏掉。原因包括：

- basketball 在图像中像素面积很小；
- 快速传球和投篮产生 motion blur；
- 球经常被手、身体或背景广告遮挡；
- 训练集只有 400 帧，ball positive samples 相对稀少。

因此，demo 中 ball 使用 short missing tolerance：检测不到时保留上一帧位置数帧并逐渐淡出。这是 visualisation-level smoothing，不代表真实 tracking。

### 6.3 Projection Error from Jumping and Airborne Objects

Homography 假设点位于同一 ground plane。player foot-point 通常满足这一近似，但当球员跳起、抬脚、被遮挡或 bbox bottom 包含阴影/鞋外区域时，bottom-centre 不再严格对应真实脚点，会造成 mini-map projection drift。ball 更明显，因为篮球常在空中，使用 bbox centre 投影到地面只是一种 approximate visual cue。

### 6.4 Camera Motion and Manual Calibration

本方法假设 camera 近似固定。如果 `Demo Video.mp4` 中存在轻微 pan/zoom/shake，固定 homography `H` 会产生系统性偏移。由于作业重点是手动 homography 和 BEV pipeline，本项目没有做 dynamic camera calibration。实际 broadcast 系统若要更稳定，需要 court line detection、frame-wise homography update 或 video stabilisation。

### 6.5 Lighting and Shadows

体育馆灯光、地板反光和球员阴影会影响 detector confidence。阴影可能导致 bbox bottom 偏低，从而影响 foot-point；广告牌和场边人员也可能与 `others` 类混淆。当前模型没有使用复杂 augmentation，这使 baseline 更易解释，但也降低了对跨场景光照变化的鲁棒性。

## 7. Conclusion

本项目完成了一个从 broadcast video 到 BEV mini-map 的完整 pipeline：YOLOv8n detector 负责多目标检测，手动 DLT/SVD homography 负责 image-to-court mapping，lightweight tracking 负责 player/referee ID 和短轨迹，最终将 detection results 与 enlarged mini-map 以 picture-in-picture 形式叠加到 `Demo Video.mp4` 上。

实验结果显示，player 和 referee 检测足以支撑小地图可视化，但 ball 和 others 的泛化性能较弱。Homography 方法清晰、可解释，适合作为 coursework proof-of-concept；其主要限制来自平面假设、手动参考点误差、镜头轻微运动、跳跃造成的 projection error，以及小目标篮球的检测困难。

最终系统展示的是一个 practical and interpretable 2D BEV tactical map，而不是精确的 3D measurement system。对于课程要求而言，它覆盖了 video ingestion, detection, coordinate transformation, mini-map rendering 和 critical evaluation 的完整流程。
